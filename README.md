# Physio Daily Tracker

> Centralized physiotherapy session tracking system for 5 centers in Iraq.
> نظام مركزي لمتابعة جلسات العلاج الطبيعي في 5 مراكز بالعراق.

---

## English

### Stack

- **Frontend:** Vite + React 18 + TypeScript + TailwindCSS (RTL Arabic primary)
- **Backend:** Express + TypeScript
- **ORM:** Drizzle ORM with `drizzle-kit` migrations
- **Database:** PostgreSQL (Neon, eu-central-1)
- **Auth:** session-based via `express-session` + `connect-pg-simple`
- **Validation:** Zod (shared between client and server)
- **Deployment:** Render (Frankfurt) — single web service serving API and built client

### Monorepo layout

```
/client        Vite React app
/server        Express API
/shared        schema.ts, validators.ts (shared types and Zod)
/migrations    Drizzle migration SQL
/scripts       seed.ts
```

### Local setup

1. Install Node 20+ and PostgreSQL (or use a Neon database).
2. Copy env file and fill in values:
   ```bash
   cp .env.example .env
   ```
3. Install dependencies:
   ```bash
   npm install
   ```
4. Run migrations:
   ```bash
   npm run db:migrate
   ```
5. Seed centers, devices, and the admin user:
   ```bash
   npm run seed
   ```
6. Start dev servers (API on `:3000`, Vite on `:5173` with proxy):
   ```bash
   npm run dev
   ```

Default admin credentials after seeding: `admin / ChangeMe!2026` — you'll be forced to change the password on first login.

### Migrations

Migrations live in `/migrations` and are managed by `drizzle-kit`:

```bash
# After editing shared/schema.ts, generate a new migration:
npm run db:generate

# Apply pending migrations to the database in DATABASE_URL:
npm run db:migrate
```

Do **not** use `db:push` — only `generate` + `migrate`.

### Production build

```bash
npm run build   # builds client/dist and dist/server
npm run start   # serves API + static client on PORT (default 3000)
```

### Deploying to Render

1. Push this repo to GitHub.
2. Create a Neon database in `eu-central-1` and copy the pooled connection string (`?sslmode=require`).
3. In Render, click **New → Blueprint** and point it at this repo. Render reads `render.yaml`.
4. In the Render dashboard, set the env vars:
   - `DATABASE_URL` — your Neon pooled connection string
   - `SESSION_SECRET` — auto-generated by Render (or set your own)
   - `NODE_ENV` — `production` (already in blueprint)
5. Click **Deploy**.
6. After the first deploy, open a Render shell and run:
   ```bash
   npm run db:migrate
   npm run seed
   ```

### Environment variables

| Variable         | Required | Notes                                                                |
| ---------------- | -------- | -------------------------------------------------------------------- |
| `DATABASE_URL`   | yes      | Postgres connection string. Include `?sslmode=require` for Neon.     |
| `SESSION_SECRET` | yes      | Long random string used to sign session cookies. 64+ random bytes.   |
| `NODE_ENV`       | yes      | `development` locally, `production` on Render.                       |
| `PORT`           | no       | Express listen port. Render sets this; default `3000` locally.       |

### Roles & permissions

Enforced server-side in addition to UI gating:

- **admin** — full CRUD on centers, devices, users; targets for any center; view/edit any session.
- **manager** — scoped to own `center_id`. Edit sessions and targets for own center. Cannot create users/devices.
- **reception** — scoped to own `center_id`. Create/edit today's sessions only; cannot edit sessions older than 24h. Read-only target progress.

## العربية

### نظرة عامة

هذا النظام يتيح للمراكز الخمسة (الناصرية، بغداد، كربلاء، الموصل، كركوك) تسجيل جلسات العلاج الطبيعي اليومية على 15 جهازاً، مع متابعة الأهداف الشهرية لكل مركز وجهاز.

### التشغيل المحلي

1. ثبّت Node 20 أو أحدث.
2. انسخ ملف البيئة:
   ```bash
   cp .env.example .env
   ```
3. ثبّت الحزم:
   ```bash
   npm install
   ```
4. شغّل ملفات الترحيل (Migrations):
   ```bash
   npm run db:migrate
   ```
5. أضف البيانات الأولية (مراكز، أجهزة، مستخدم النظام):
   ```bash
   npm run seed
   ```
6. شغّل البيئة التطويرية:
   ```bash
   npm run dev
   ```

بيانات دخول مدير النظام بعد الـSeed: `admin / ChangeMe!2026` — سيُطلب منك تغيير كلمة المرور عند أول تسجيل دخول.

### النشر على Render

1. ارفع المستودع إلى GitHub.
2. أنشئ قاعدة بيانات على Neon في منطقة `eu-central-1` وانسخ سلسلة الاتصال.
3. في Render اختر **New → Blueprint** وأشر إلى هذا المستودع.
4. عيّن المتغيرات في لوحة Render:
   - `DATABASE_URL`
   - `SESSION_SECRET` (يولّده Render تلقائياً)
   - `NODE_ENV=production`
5. اضغط **Deploy**.
6. بعد أول نشر، افتح shell على Render وشغّل:
   ```bash
   npm run db:migrate
   npm run seed
   ```

### الصلاحيات

- **مدير النظام (admin)**: تحكم كامل بالمراكز والأجهزة والمستخدمين والأهداف.
- **مدير المركز (manager)**: محدود بمركزه فقط — تعديل الجلسات والأهداف.
- **استقبال (reception)**: إدخال جلسات اليوم لمركزه فقط، ولا يستطيع تعديل ما مضى عليه أكثر من 24 ساعة.
